# Tela 07 — IA & Automação

**Rota:** `/saas/ia`
**Arquivo:** `layers/saas/pages/saas/ia.vue`
**Composable principal:** `useAI()` — `layers/saas/composables/useAI.ts`

## Visão Geral

Central de configuração e monitoramento do agente de IA. Permite configurar o comportamento do agente, gerenciar base de conhecimento, criar fluxos de conversa e automações baseadas em eventos.

## Tabs

| Tab | Conteúdo |
|-----|---------|
| `painel` | Dashboard com métricas de conversas IA, heatmap de atividade, FAQs não resolvidas |
| `agente` | Configuração completa do agente (nome, tom, mensagens, capacidades) |
| `conhecimento` | CRUD da base de conhecimento (FAQs, regras, procedimentos) |
| `fluxos` | Fluxos de conversa pré-definidos (ex: "Agendar", "Consultar preço") |
| `automacoes` | Automações baseadas em eventos (ex: "Confirmar agendamento 2h antes") |

## Tipos

```typescript
interface AIAgentConfig {
  name: string
  description: string
  avatarUrl: string
  tone: 'professional' | 'friendly' | 'casual' | 'formal' | 'custom'
  customToneInstructions: string
  useEmojis: boolean
  emojiFrequency: 'never' | 'rarely' | 'sometimes' | 'often'
  language: string
  autoDetectLanguage: boolean
  welcomeMessage: string
  offHoursMessage: string
  maxMessagesBeforeTransfer: number
  alwaysTransferTopics: string[]
  aiSchedule: object              // horários em que a IA está ativa
  capabilities: AICapability[]
}

interface AICapability {
  key: string                     // ex: 'scheduling', 'pricing', 'faq'
  label: string
  description: string
  enabled: boolean
}

interface KnowledgeEntry {
  id: string
  title: string
  category: 'FAQ' | 'Regras' | 'Procedimentos' | 'Produtos' | 'Promoções' | 'Outro'
  content: string
  exampleQuestion: string
  active: boolean
  createdAt: string
}

interface ConvFlow {
  id: string
  name: string
  description: string
  trigger: string                 // ex: 'agendar', 'preço', 'localização'
  active: boolean
  usageCount: number
  completionRate: number          // percentual 0-100
}

interface BusinessAutomation {
  id: string
  name: string
  description: string
  triggerType: string             // ex: 'before_appointment', 'after_appointment'
  triggerLabel: string
  actionType: string              // ex: 'whatsapp_message', 'email'
  actionLabel: string
  active: boolean
  stats: { sent: number, delivered: number }
}

interface AIDashboard {
  todayConversations: number
  resolvedByAI: number
  aiResolutionRate: number        // percentual
  transferredToHuman: number
  avgResponseSeconds: number
  dailyData: Array<{ date: string, total: number, resolved: number }>
  heatmapData: number[][]         // [dia_semana][hora] → volume
}
```

## Funções do Composable

| Função | Descrição |
|--------|-----------|
| `fetchAll()` | Carrega config, dashboard, conhecimento, fluxos, automações |
| `saveAgentConfig(config)` | Salva configuração do agente (mock) |
| `addKnowledgeEntry(entry)` | Adiciona entrada na base de conhecimento |
| `updateKnowledgeEntry(id, data)` | Atualiza entrada |
| `toggleKnowledgeEntry(id)` | Ativa/desativa entrada |
| `toggleAutomation(id)` | Ativa/desativa automação |
| `toggleFlow(id)` | Ativa/desativa fluxo |

## Componentes Especiais

- **Heatmap** (tab Painel): grade 7×24 com opacidade proporcional ao volume de conversas por hora/dia. Implementado com divs e CSS vars.
- **`ZimaPhonePreview`** (tab Agente): preview em tempo real das mensagens de boas-vindas e pós-horário enquanto o usuário edita.

## Rota dinâmica

`/saas/ia/fluxos/[id]` — editor de fluxo detalhado (canvas simplificado de nós).
