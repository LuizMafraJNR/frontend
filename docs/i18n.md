# Internacionalização (i18n)

## Configuração

- **Locale padrão**: `pt-BR` (sem prefixo na URL)
- **Locale secundário**: `en` (prefixo `/en/`)
- **Estratégia**: `prefix_except_default`
- **Arquivos de tradução**: `i18n/pt-BR.json`, `i18n/en.json` (carregamento lazy)

## Uso em Componentes

```vue
<script setup lang="ts">
const { t, locale } = useI18n()
</script>

<template>
  <button>{{ t('common.save') }}</button>
  <p>{{ t('patients.title') }}</p>
</template>
```

## Navegação com Locale

Sempre usar `useLocalePath()` para gerar URLs:

```typescript
const localePath = useLocalePath()

// Redirecionar com locale correto
navigateTo(localePath('/pacientes'))

// Link em template
<NuxtLink :to="localePath('/agendamentos')">Agendamentos</NuxtLink>
```

## Datas e Números

Usar `useI18n()` para formatar datas e números respeitando o locale:

```typescript
const { d, n } = useI18n()

d(new Date(), 'short')      // "29/03/2026" (pt-BR) | "3/29/2026" (en)
n(1234.56, 'currency')      // "R$ 1.234,56" (pt-BR) | "$1,234.56" (en)
```

## Adicionando uma Chave de Tradução

1. Adicionar a chave em `i18n/pt-BR.json` (pt-BR é a fonte da verdade)
2. Adicionar chave equivalente em `i18n/en.json`
3. **Nunca** hardcodar texto visível ao usuário nos templates

## Adicionando um Novo Idioma

1. Criar `i18n/<code>.json` com todas as chaves de `pt-BR.json`
2. Adicionar entrada ao array `i18n.locales` em `nuxt.config.ts`
3. Atualizar seletor de idioma no header
4. Atualizar este arquivo

## SEO Multilíngue

O `@nuxtjs/i18n` injeta automaticamente `<link rel="alternate" hreflang>` nas páginas.
Usar `useSeoMeta()` com `useI18n()` para títulos e descrições localizadas:

```typescript
const { t } = useI18n()
useSeoMeta({ title: t('patients.title') })
```
